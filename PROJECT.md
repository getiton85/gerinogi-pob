# 게리노기 지멋대로 POB 밸류

마비노기 모바일 검술사와 대검전사 세팅을 룬, 패시브, 스탯, 조건부 효과 기준으로 비교하는 정적 웹앱입니다.

## 현재 원칙

- 기존 기능 유지 후 새 기능 추가
- 룬 밸류는 현재 장착 세팅 대비 최종 기대딜 변화율 기준
- 속도, 쿨타임, 회복 계열은 룬 추천 점수에 직접 가산하지 않고 스킬별 쿨/1분 피해에서 분리 표시
- UI에는 유틸 옵션을 표시할 수 있지만 추천 순위는 기대딜 기준으로 정렬
- 임시 데이터와 의미 없는 이름은 배포본에 남기지 않음

## 현재 주요 파일

- index.html: 앱 진입점
- style.css: 화면 배치와 반응형 스타일
- data.js: 룬, 직업, 패시브 DB
- core.js: 스탯 환산, 룬 효과 합산, 기대딜 밸류 계산, 자체 검수
- app.js: 장착 UI, 룬 선택창, 대시보드, 추천 순위 렌더링

## GitHub Pages 배포

이 앱은 별도 서버 없이 브라우저에서 실행되는 정적 웹앱입니다. 저장소 루트에 현재 파일들을 올리고 GitHub Pages를 저장소 루트 또는 배포 브랜치로 지정하면 웹에서 볼 수 있습니다.

# 버전 표시 규칙

- 배포되는 기능, 데이터, 계산식, UI가 바뀔 때마다 좌측 상단 버전 숫자를 반드시 올린다.
- 기준 파일은 `version.json`이며, 화면 표시는 `index.html`의 `#appVersion`이다.
- `style.css`, `data.js`, `core.js`, `app.js` 참조에는 같은 버전 쿼리를 붙여 GitHub Pages/브라우저 캐시를 줄인다.
- Codex는 수정 작업을 완료하기 전에 버전 증가 여부를 검수한다.


## v0.0004 계산식 v2

- 공식 A~L 구조를 분리한 `formulaV2Context`를 추가했다.
- 검술사 스킬 DB를 추가하고 노크리/크리/브레이크/브레이크 익스텐션 예상 피해를 표시한다.
- 밤의 축복 평균 유지율 기본값은 15초/60초 기준 0.25로 둔다.
- 스킬계수 원본이 없는 스킬은 스크린샷 툴팁 피해를 기준으로 계산한다.

## v0.0027 대검전사

- 검술사와 대검전사를 동시에 적용할 수 없는 단일 직업 선택 구조로 변경했다.
- 대검전사 액티브 스킬, 각인/폼, 기본·시즌 패시브를 사용자 인수 문서 기준으로 추가했다.
- 차지 단계별 피해는 합산하지 않고 각 단계의 선택 피해로 분리 표시한다.
- 공통 시즌 패시브 `깊어지는 어둠`은 `1 + clamp(레벨, 0, 150) × 0.001` 최종 피해 배율을 사용한다.
- 재사용 대기 시간이 제공되지 않은 스킬은 임의의 쿨타임이나 1분 피해를 만들지 않는다.

## v0.0028 직업별 독립 스킬 패널

- 검술사와 대검전사 스킬 예상 데미지를 각각 독립된 접이식 패널로 표시한다.
- 두 패널은 패시브 OFF 상태에서도 기본 스탯과 장착 룬 기준 예상 피해를 계속 표시한다.
- ON인 직업 패널만 해당 직업 패시브를 적용하며, 두 직업은 동시에 ON으로 설정할 수 없다.

## v0.0029 밸류점수 수식 통합

- v16의 기준 세팅 100.00 정규화와 v23의 퍼센트 단위 보정을 유지한다.
- 대시보드, 후보 룬, 비교 세팅이 모두 `calc.score` 단일 기대딜 계산 결과를 사용한다.
- 후보와 비교 세팅의 현재 대비 변화율은 `(후보 ÷ 현재 - 1) × 100`으로 계산한다.
- 스탯창 공격력에 이미 포함된 기준 룬의 고정 공격력과 공격력%는 후보 계산에서 차이만 반영한다.
- 궁극기 피해, 브레이크 추가 피해, 콤보 피해를 해당 기대딜 축에 포함하고 중복 계산하던 별도 공식을 제거한다.
