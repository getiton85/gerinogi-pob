# GitHub 연동 작업 안내

## 현재 웹 주소

https://getiton85.github.io/gerinogi-pob/

## 현재 GitHub 저장소

https://github.com/getiton85/gerinogi-pob

## GitHub Desktop 다운로드

https://desktop.github.com/

## 추천 작업 흐름

1. GitHub Desktop 설치
2. GitHub Desktop에서 GitHub 계정 로그인
3. File > Clone repository 선택
4. GitHub.com 탭에서 getiton85/gerinogi-pob 선택
5. Local path는 Codex가 접근 가능한 폴더로 지정

추천 위치:

```text
C:\Users\freem\Documents\Codex\gerinogi-pob
```

6. Clone 클릭
7. 이후 Codex에게 위 폴더를 기준으로 작업해달라고 말하기
8. Codex가 수정/검수 완료
9. GitHub Desktop에서 변경사항 확인
10. Summary에 업데이트 내용을 적고 Commit 클릭
11. Push origin 클릭
12. GitHub Pages가 자동 반영
13. 외부 사용자는 새로고침

## 업데이트할 때 말하면 좋은 문장

```text
C:\Users\freem\Documents\Codex\gerinogi-pob 폴더가 GitHub와 연결된 최신 작업 폴더야.
PROJECT.md를 먼저 읽고 기존 기능 유지하면서 작업해줘.
수정 후 7단계 검수까지 하고 GitHub Desktop에서 커밋할 수 있게 정리해줘.
```

## 주의

- GitHub Desktop은 파일을 자동으로 웹에 올리는 도구입니다.
- Codex가 코드를 수정하고, 사용자가 GitHub Desktop에서 Commit + Push를 누르는 방식이 가장 안정적입니다.
- Push 후 GitHub Pages 반영까지는 보통 30초~3분 걸립니다.
- 웹이 예전 화면이면 새로고침 또는 강력 새로고침을 합니다.
